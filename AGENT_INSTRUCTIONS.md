# Архитектура и Инструкции по проекту BotStudio (ex-antifreelance)

**ВНИМАНИЕ ДЛЯ ВСЕХ ИИ-АССИСТЕНТОВ:** Этот файл является главным источником истины по архитектуре проекта. При любых изменениях структуры, добавлении новых разделов или технологий — **ОБЯЗАТЕЛЬНО** обновляйте этот файл и делайте коммит в GitHub.

## 1. Описание проекта
Сайт-визитка и блог (База знаний) для агентства **BotStudio**, занимающегося разработкой умных Telegram-ботов с локальным AI для бизнеса.
- **Стек:** Astro (v5+), Vanilla CSS (с CSS Variables), GSAP (анимации).
- **Размещение:** Статический сайт (SSG).

## 2. Структура проекта
- `src/pages/index.astro` — Главная страница (Landing Page).
- `src/pages/knowledge/index.astro` — Главная страница Базы знаний (список статей).
- `src/pages/knowledge/[slug].astro` — Динамический роут для отображения отдельной статьи из Базы знаний.
- `src/content/knowledge/` — Директория с Markdown (`.md`) файлами статей.
- `src/content/config.ts` — Конфигурация Astro Content Collections (Zod-схемы для валидации frontmatter статей).
- `src/components/` — UI-компоненты (Header, Hero, Cases, Reviews, CTA, Portfolio).
- `src/layouts/Layout.astro` — Базовый HTML-шаблон, подключение шрифтов (Bebas Neue, Inter) и глобальных стилей.
- `src/styles/global.css` — Глобальные стили, CSS-переменные (цвета: `--black`, `--white`, `--orange`, `--lime`), базовые анимации (например, `@keyframes auto-press` для кнопок).

## 3. Ключевые архитектурные решения
### 3.1. База знаний (Блог)
Реализована через **Astro Content Collections**.
- **Почему:** Обеспечивает строгую типизацию (Zod), автоматический роутинг и удобное написание контента в Markdown.
- **Как добавить статью:** Создать `.md` файл в `src/content/knowledge/` с обязательным frontmatter: `title`, `description`, `pubDate`, `tags`.

### 3.2. Анимации (GSAP)
- Используется GSAP и ScrollTrigger (подключаются через CDN в `Layout.astro`).
- Класс `.reveal` используется для элементов, которые должны плавно появляться при скролле (настроено в `Layout.astro`).
- Кнопки (`.btn-primary`) используют CSS-анимацию `auto-press` (пульсация тени каждые 3 секунды) вместо стандартного hover-увеличения.

### 3.3. Портфолио (Кейсы)
- Блок портфолио (`Portfolio.astro`) использует паттерн **Scroll-Linked Device Frames**.
- Внутри мокапов устройств (телефон/ноутбук) контент (скриншоты/видео) прокручивается синхронно со скроллом страницы с помощью GSAP ScrollTrigger.

## 4. Правила работы (Workflow)
1. **Документируй всё:** Любое изменение архитектуры (новый роут, новая коллекция, новый сложный компонент) должно быть описано в этом файле.
2. **GitHub Issues:** Каждая крупная задача должна быть оформлена как Issue в GitHub (используй `gh issue create`).
3. **Коммиты:** Делай осмысленные коммиты после каждого логического шага.
4. **Mobile-First:** Все компоненты должны корректно отображаться на мобильных устройствах. Используй `clamp()` для шрифтов и отступов, проверяй CSS Grid на узких экранах.
5. **Без лишних библиотек:** Используй встроенные возможности Astro и Vanilla CSS. Избегай добавления тяжелых JS-фреймворков (React/Vue), если это не абсолютно необходимо.
